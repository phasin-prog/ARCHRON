# ARCHRON Visual Operating System (AVOS)

**Version:** 1.0  
**Status:** Living Design Constitution  
**Encoding:** UTF-8  
**Language Strategy:** Thai-First Design System (อธิบายหลักการและระบบสุนทรียะเป็นภาษาไทยควบคู่กับคำศัพท์เฉพาะทางสากล)

---

## 01. Philosophy (ปรัชญาพื้นฐาน)

ARCHRON ไม่ใช่เพียงเว็บไซต์ (ARCHRON is not a website.)  
แต่ ARCHRON คือ **ระบบความรู้ที่มีชีวิต (Living Knowledge System)**

อินเทอร์เฟซไม่ควรกลายเป็นจุดศูนย์กลางความสนใจของผู้คน  
หากแต่ **"เนื้อหาและความรู้"** ต่างหากที่ต้องโดดเด่นที่สุดในสายตาผู้อ่าน  
ทุกการตัดสินใจทางภาพและการออกแบบ (Visual Decisions) ต้องทำหน้าที่ลดอุปสรรคในการอ่าน (Reduce friction) เพิ่มพูนความเข้าใจ (Increase understanding) และตอกย้ำความกระหายรู้ (Reinforce curiosity)

> **"Beauty exists to reveal meaning. Never to replace it."**  
> ความงามมีอยู่เพื่อเผยให้เห็นความหมาย หาใช่เพื่อมาแทนที่ความหมายไม่

## 02. Design Principles (หลักการออกแบบ 7 ประการ)

### 2.1 Human First (ยึดมนุษย์เป็นศูนย์กลาง)
ออกแบบโดยคำนึงถึงนักอ่านและผู้แสวงหาปัญญา ไม่ใช่ออกแบบเพื่อเอาใจนักออกแบบด้วยกันเอง

### 2.2 Knowledge First (เนื้อหาความรู้อยู่เหนือสิ่งอื่นใด)
คุณค่าของเนื้อหาสำคัญกว่าเครื่องแต่งแต้มหรือลูกเล่นตกแต่งเสมอ

### 2.3 Quiet Interface (อินเทอร์เฟซที่เงียบสงบ)
ลดสิ่งรบกวนทางสายตา (Visual Noise) เพื่อเพิ่มพลังสมาธิและการจดจ่อระดับสูงสุด

### 2.4 Timeless Design (ความเหนือกาลเวลา)
หลีกเลี่ยงงานดีไซน์ตามกระแสแฟชั่นฉาบฉวย มุ่งสู่ความเรียบหรูที่ยั่งยืนผ่านกาลเวลา

### 2.5 Functional Beauty (ความงามที่เปี่ยมคุณค่าใช้สอย)
องค์ประกอบที่งดงามทุกชิ้นต้องมีเป้าหมายหน้าที่ในการสื่อสารที่ชัดเจน

### 2.6 Progressive Complexity (ซับซ้อนอย่างเป็นลำดับขั้น)
เรียบง่ายในสัมผัสแรกเห็น แต่ทรงพลังและลุ่มลึกเมื่อผู้ใช้ต้องการสืบค้นต่อ

### 2.7 Invisible Technology (เทคโนโลยีที่ล่องหน)
ผู้ใช้งานต้องได้สัมผัสกับประสบการณ์แห่งปัญญาโดยไม่รู้สึกว่ากำลังถูกบังคับด้วยเครื่องมือซอฟต์แวร์

## 04. Visual Personality (บุคลิกภาพทางสายตา)

บุคลิกของ ARCHRON ต้องยึดถือคุณลักษณะด้านล่างอย่างเคร่งครัด:

### ✅ ลักษณะที่ต้องสะท้อนออกมา (Embrace)
* **Elegant** (สง่างาม ประณีต)
* **Academic** (น่าเชื่อถือทางวิชาการ)
* **Warm** (อบอุ่น นุ่มนวล)
* **Calm** (สงบเยือกเย็น)
* **Curious** (ชวนให้สงสัยใคร่รู้)
* **Intellectual** (ลุ่มลึกทางปัญญา)
* **Organic** (กลมกลืนเป็นธรรมชาติ)
* **Minimal** (เรียบง่ายแต่ทรงพลัง)
* **Sophisticated** (มีระดับ วางตัวดี)
* **Timeless** (ยั่งยืนเหนือกาลเวลา)

### ❌ ลักษณะที่ห้ามมีเด็ดขาด (Never)
* **Corporate** (องค์กรธุรกิจจ๋าสีฟ้าตัดขาวแข็งกระด้าง)
* **Cold** (เย็นชา ไร้ชีวิตจิตใจ)
* **Gaming / Sci-Fi / Cyberpunk** (ฉูดฉาดแบบเกมเมอร์ นีออนสะท้อนแสง)
* **Glass Everywhere** (กระจกฝ้าซ้อนกันจนลายตาและอ่านยาก)
* **Over Decorative** (ประดับประดาฟุ่มเฟือยไร้เหตุผล)

## 05. Color Engine (ระบบขับเคลื่อนสีตามศาสตร์ความรู้)

### Base Palette (กลุ่มสีพื้นฐานแห่งคลังปัญญา)
* **Canvas:** โทนสีเข้มลุ่มลึกสำหรับพื้นหลังหลัก (`--color-deep-navy` / `#080B16`)
* **Warm Ivory:** สีงาช้างอบอุ่นสำหรับตัวอักษรหลัก ลดความล้าของสายตา
* **Paper / Stone:** สีของผิวกระดาษและหินคลาสสิกสำหรับกรอบและการ์ดเนื้อหา
* **Graphite:** สีเทาเข้มขรึมสำหรับเส้นแบ่งเขตและเงา

### Dynamic Domain Accents (สีแต้มตามสาขาวิชาความรู้)
สีแต้ม (Accent Color) ปรับเปลี่ยนอัตโนมัติตามหมวดหมู่เนื้อหา โดยยังคงอัตลักษณ์โดยรวมไว้:
* **Psychology (จิตวิทยา):** *Royal Violet* (ม่วงราชสำนักอันลึกลับแห่งจิตใต้สำนึก)
* **Philosophy (ปรัชญา):** *Indigo* (ครามลุ่มลึกแห่งการขบคิดสัจธรรม)
* **History (ประวัติศาสตร์):** *Bronze* (สำริดโบราณแห่งกาลเวลาที่ผ่านพ้น)
* **Religion (ศาสนศาสตร์):** *Crimson* (แดงชาดแห่งศรัทธาและพิธีกรรมศักดิ์สิทธิ์)
* **Mythology (ปรัมปราวิทยา):** *Amber* (อำพันเรืองรองแห่งตำนานและปกรณัม)
* **Anthropology (มานุษยวิทยา):** *Terracotta* (ดินเผาอบอุ่นแห่งอารยธรรมมนุษย์)
* **Linguistics (ภาษาศาสตร์):** *Emerald* (มรกตแห่งวิวัฒนาการภาษาและถ้อยคำ)
* **Neuroscience (ประสาทวิทยา):** *Cyan* (ฟ้าครามสว่างแห่งกระแสประสาทและการรับรู้)
* **Artificial Intelligence (ปัญญาประดิษฐ์):** *Azure* (ครามสว่างแห่งโครงข่ายปัญญาสังเคราะห์)

## 06. Surface Engine (ระบบพื้นผิวและมิติซ้อนทับ)

ลำดับชั้นของพื้นผิวจากล่างขึ้นบน เพื่อสร้างมิติที่นุ่มนวลและเป็นระเบียบ:
1. **Background:** พื้นหลังสุดลึกสุดของจักรวาล ARCHRON
2. **Canvas:** กระดานอ่านเนื้อหาหลัก
3. **Surface:** พื้นผิวรองรับส่วนแบ่งเนื้อหา
4. **Panel:** แผงควบคุมและส่วนนำทางด้านข้าง
5. **Card:** การ์ดบรรจุความรู้แต่ละชิ้น
6. **Floating Layer:** ชั้นลอยตัวสำหรับปุ่มนำทางด่วน (เช่น Scroll to top)
7. **Modal / Overlay:** หน้าต่างโต้ตอบเฉพาะกิจที่ลอยเหนือเนื้อหา
8. **Tooltip:** ป้ายอธิบายสั้นที่สุดเมื่อวางเมาส์เหนือจุดสำคัญ

> **กฎเหล็ก:** พื้นผิวต้องให้ความรู้สึกคล้าย **กระดาษคุณภาพสูงปูบนแท่นจัดแสดงพิพิธภัณฑ์** หลีกเลี่ยงการใช้สีดำสนิท (Pure Black `#000000`) และขาวสนิท (Pure White `#FFFFFF`)

## 07. Typography System (ระบบตัวอักษรและการอ่าน)

แบ่งการใช้งานออกเป็น 3 ตระกูลหลัก เพื่อสร้างลำดับชั้นที่ชัดเจนและสบายตา:
* **Editorial (Noto Serif Thai):** สำหรับหัวเรื่องหลัก (`<h1>` - `<h3>`) และคำโปรย ให้ความรู้สึกสง่างามแบบสิ่งพิมพ์วิชาการ
* **UI (IBM Plex Sans Thai):** สำหรับเนื้อหาบทความขนาดยาว (Body), ป้ายนำทาง, ปุ่มกด และ Interface ทั้งหมด อ่านง่ายสบายตาที่สุด
* **Mono (JetBrains Mono / Code Font):** สำหรับบล็อกโค้ด, เลขบรรณานุกรม, เชิงอรรถ (Citation/Reference) และเมตาข้อมูล

> **Reading Rhythm:** ระยะห่างบรรทัด (Line Height) ต้องกว้างโปร่งสบาย ไม่บีบอัด ให้สายตากวาดอ่านเนื้อหาต่อเนื่องหลายพันคำได้อย่างผ่อนคลาย

---

## 08. Grid System (ระบบกริดและการจัดวาง)

* **Desktop:** โครงสร้างกริด 12 คอลัมน์ (12 Columns)
* **Tablet:** โครงสร้างกริด 8 คอลัมน์ (8 Columns)
* **Mobile:** โครงสร้างกริด 4 คอลัมน์ (4 Columns)
* **Rhythm Spacing:** ใช้จังหวะ **8pt Grid** (8px, 16px, 24px, 32px, 48px, 64px) ทุกองค์ประกอบสอดประสานกันอย่างเป็นระเบียบ

---

## 09. Icon System (ระบบไอคอนเชิงสัญลักษณ์)

ไอคอนคือ **สัญลักษณ์สื่อความหมาย (Symbolic)** ไม่ใช่ของตกแต่งสวยงาม:
* ขนาดมาตรฐานบนกริด: `16px`, `20px`, `24px`, `32px`, `48px`, `64px`, `96px`
* รูปแบบไฟล์: **SVG Stroke Only** (ใช้เส้นขอบเป็นหลัก)
* คุณสมบัติเส้น: *Round Join*, *Round Cap*, ความโค้งมุมและความหนาเส้นสม่ำเสมอเท่ากันทั้งระบบ
* หลีกเลี่ยงไอคอนทึบ (Filled Icons) ยกเว้นใช้เพื่อบ่งบอกสถานะการเลือก (Active State)

* **Article (บทความ):** *Notebook* (สมุดบันทึกการวิเคราะห์)
* **Concept (มโนทัศน์):** *Crystal* (ผลึกปัญญาที่กลั่นกรองแล้ว)
* **Thinker (นักคิด):** *Bust* (รูปปั้นครึ่งตัวเกียรติยศ)
* **School (สำนักคิด):** *Temple* (วิหารแห่งสายธารปัญญา)
* **Theory (ทฤษฎี):** *Orbit* (วงโคจรแห่งกฎเกณฑ์)
* **Book (หนังสือ):** *Open Book* (ตำราเปิดกว้าง)
* **Timeline (เส้นเวลา):** *Hourglass* (นาฬิกาทรายแห่งกาลเวลา)
* **Reference (แหล่งอ้างอิง):** *Bookmark* (ที่คั่นหน้าจดจำต้นฉบับ)
* **Collection (คลังความรู้):** *Archive Box* (กล่องจดหมายเหตุ)
* **Language (ภาษา):** *Scroll* (ม้วนคัมภีร์โบราณ)
* **Civilization (อารยธรรม):** *Column* (เสาหินอารยธรรม)
* **Religion (ศาสนา):** *Sacred Circle* (วงกลมศักดิ์สิทธิ์)
* **Mythology (ตำนาน):** *Labyrinth* (เขาวงกตปริศนา)
* **Psychology (จิตวิทยา):** *Mind* (ดวงจิตและโครงข่ายรับรู้)
* **Philosophy (ปรัชญา):** *Compass* (เข็มทิศนำทางสัจธรรม)
* **Anthropology (มานุษยวิทยา):** *Tribe* (สัญลักษณ์ผู้คนและชนเผ่า)
* **Neuroscience (ประสาทวิทยา):** *Neuron* (เซลล์ประสาทเชื่อมโยง)

## 11. Motion Engine (ระบบการเคลื่อนไหวที่มีเป้าหมาย)

การเคลื่อนไหวมีไว้เพื่อสื่อสาร **สถานะ (State)**, **การโฟกัส (Focus)** และ **ลำดับชั้น (Hierarchy)** เท่านั้น ห้ามใส่เพื่อความบันเทิงฉาบฉวย:
* **Motion Types:** Fade (ค่อยๆ ปรากฏ), Slide (เลื่อนเข้าออก), Expand/Collapse (กาง/พับ), Scale (ขยายขนาด), Float (ลอยตัวนุ่มนวล), Orbit (โคจรรอบจุดศูนย์กลาง), Reveal (เปิดเผยเนื้อหา)
* **Timing Duration:** Short (150ms สำหรับ UI โต้ตอบ), Normal (300ms สำหรับเปลี่ยนหน้า/เปิดแผง), Long (500ms+ สำหรับโหลดกลุ่มดาว)
* **Reduced Motion:** รองรับ `prefers-reduced-motion: reduce` เสมอ โดยเปลี่ยนจากการขยับเป็นการจางเข้า (Fade) ทันที

## 12. Component Engine (สถาปัตยกรรมคอมโพเนนต์)

โครงสร้างการสืบทอดชั้นคอมโพเนนต์จากพื้นฐานสู่ความซับซ้อน:
```text
Primitive Components (ปุ่มพื้นฐาน, ป้ายอักษร, กรอบไอคอน)
   ↓
Semantic Components (การ์ดบทความ, ช่องค้นหาพร้อมฟิลเตอร์)
   ↓
Context Components (แผงสารบัญเนื้อหา, ส่วนแนะนำแนวคิดที่เชื่อมโยง)
   ↓
Feature Components (กราฟ Constellation, สภาพแวดล้อมตัวเขียน Studio)
   ↓
Page Components (หน้าแสดงผลสมบูรณ์แต่ละ Route)
```
ทุกคอมโพเนนต์สืบทอดดีเอ็นเอจากฐานรากเดียวกัน และปรับตัวเข้ากับบริบทหน้าจออย่างกลมกลืน

---

## 13. Dynamic Context Engine (บรรยากาศตามบริบทความรู้)

เลย์เอาต์โครงสร้างหลักคงเดิม แต่อารมณ์และบรรยากาศปรับเปลี่ยนตามสาขาวิชา:
* **Psychology:** *Organic, Soft, Reflective* (นุ่มนวล สะท้อนตัวตน เป็นธรรมชาติ)
* **History:** *Museum, Ancient, Layered* (ประหนึ่งพิพิธภัณฑ์โบราณที่มีชั้นหินประวัติศาสตร์)
* **Philosophy:** *Geometric, Balanced, Minimal* (เรขาคณิตสมดุล สมมาตร และเรียบง่าย)
* **Religion:** *Sacred, Light, Symbolic* (แสงสว่าง เรืองรอง เปี่ยมสัญลักษณ์)
* **Mythology:** *Epic, Textured, Illustrative* (มหากาพย์ มีพื้นผิวกระดาษโบราณ และภาพประกอบลุ่มลึก)
* **Science:** *Precise, Structured, Clean* (แม่นยำ เป็นระบบ และสะอาดตาขั้นสุด)

## 14. Card System (ระบบการ์ดความรู้เฉพาะทาง)

การ์ดแต่ละประเภทจัดวางสาระสำคัญ (Visual Hierarchy) แตกต่างกันตามธรรมชาติของข้อมูล:
* **Article Card:** *Reading First* (เน้นชื่อเรื่อง คำโปรย และเวลาที่ใช้อ่าน)
* **Book Card:** *Cover First* (เน้นภาพหน้าปก ทรงคุณค่า และผู้แต่ง)
* **Thinker Card:** *Portrait First* (เน้นภาพวาด/รูปปั้นนักคิดและช่วงปีประวัติศาสตร์)
* **Concept Card:** *Definition First* (เน้นคำนิยามสั้น และสายธารสำนักคิดที่สังกัด)
* **Timeline Card:** *Chronology First* (เน้นตัวเลขปีศักราชและลำดับก่อนหลัง)
* **Reference Card:** *Citation First* (เน้นข้อความอ้างอิงปฐมภูมิและแหล่งที่มา)
* **Collection Card:** *Relationship First* (เน้นจำนวนความเชื่อมโยงและภาพรวมซีรีส์)

## 15. Background Engine (ระบบพื้นหลังที่เงียบสงบ)

พื้นหลังต้องไม่แย่งความสนใจจากตัวหนังสือ:
* **Background Layers:** ลวดลายกระดาษโบราณ (Paper Texture), กริดบางเฉียบ (Grid), จุดประกายกลุ่มดาว (Constellation), เม็ดเกรนละเอียด (Noise), แสงเรืองรองเฉพาะจุด (Soft Glow), เรขาคณิตโบราณ (Ancient Geometry)
* **Controlled Opacity:** ความเข้มของลวดลายจำกัดไว้เพียง **2%, 4% หรือสูงสุดไม่เกิน 6%** ห้ามไล่สีฉูดฉาดหรือรบกวนสายตาเด็ดขาด

## 16. Illustration Language (ภาษาภาพประกอบวิชาการ)

ภาพประกอบทั้งหมดต้อง **"สอนและให้ความรู้"** (Teach, not decorate):
* รูปแบบที่อนุญาต: แผนภาพวิเคราะห์ (Editorial Diagrams), แผนที่ความรู้ (Knowledge Maps), โครงข่ายกลุ่มดาว (Constellations), เส้นเวลา (Timelines), ภาพร่างต้นฉบับโบราณ (Ancient Manuscripts), ภาพอธิบายทางวิทยาศาสตร์ (Scientific Illustrations), ป้ายคำอธิบายพิพิธภัณฑ์ (Museum Labels)

## 17. Layout Archetypes (ต้นแบบโครงสร้างเลย์เอาต์ 8 สไตล์)

1. **Article Layout:** *Magazine Style* (คอลัมน์สมดุล อ่านง่าย สบายตา)
2. **Thinker Layout:** *Biography Style* (โปรไฟล์เกียรติยศ ประวัติ ผลงาน และความสัมพันธ์)
3. **Concept Layout:** *Dictionary / Wiki Style* (นิยามกระชับ รากศัพท์ และ Backlinks)
4. **Book Layout:** *Library Style* (ข้อมูลบรรณานุกรม สรุป และบทวิจารณ์)
5. **Timeline Layout:** *Museum Walk Style* (เส้นทางเดินตามแนวนอนหรือแนวตั้งตามกาลเวลา)
6. **Search Layout:** *Explorer Style* (ผลลัพธ์ฉับไว พร้อมแผงตัวกรองอัจฉริยะ)
7. **Studio Layout:** *Workspace Style* (พื้นที่เขียนไร้สิ่งรบกวน แผงเครื่องมือด้านข้าง)
8. **Dashboard Layout:** *Control Center Style* (สรุปภาพรวมความคืบหน้าและสถิติเข้าใจง่าย)

## 18. Accessibility (มาตรฐานการเข้าถึงสำหรับทุกคน)

* ผ่านเกณฑ์ **WCAG AA+** เป็นมาตรฐานขั้นต่ำ
* รองรับการนำทางด้วยแป้นพิมพ์สมบูรณ์แบบ (Full Keyboard Navigation) พร้อมเส้นขอบโฟกัสชัดเจน
* รองรับโปรแกรมอ่านหน้าจอ (Screen Reader Friendly & ARIA Labels)
* อัตราส่วนความต่างสี (High Contrast Compliance) ชัดเจน อ่านสบายทั้งกลางวันและกลางคืน
* พื้นที่กด (Click Targets) กว้างขวาง สะดวกต่อการใช้งานบนจอสัมผัส
